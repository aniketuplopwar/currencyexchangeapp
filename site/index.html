High Level description of implementation

Below is the Design diagram to explain the design

This implementation consists of 4 high level components

  • Service
  • View
  • Model
  • App

Service

Service is responsible for interacting with server on socket and the given topic, and return updates to App. On initialization it makes a socket connection to the server using stomp and subscribe to topic '/fx/prices'

View

View is responsible build the exchange table, and add/update the table content whenever App sends an update

Model

Model is responsible to translate the messages coming from service to make it compatible to the view, Also it holds the previous values to make a list of midPrice, so that sparkline graph can be drawn on midPrice values, over last 30 seconds.

App

App is the integrating point for above 3 components. App subscribes to the service for updates. Sends update to Model to get transalated message. Calls view`s update to render the update received.
